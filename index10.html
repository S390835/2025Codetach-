Introduction & History

C++ was created by Bjarne Stroustrup (1979 onward) as "C with Classes". It supports procedural, object-oriented, and generic programming paradigms. Modern C++ (C++11/14/17/20/23) added many powerful features.

Basics & Syntax

  • Statements end with ;, blocks use { }
  • Namespaces: namespace std { } and using namespace std; (prefer std::)
  • Auto type deduction: auto x = 10;
  • Header files: <iostream>, <vector>, <string>

Types & Variables

Built-in: int, long, float, double, char, bool, void. Modifiers and fixed-width types: int32_t, uint64_t from <cstdint>.

Functions & Overloading

Function overloading, default arguments, inline, constexpr, noexcept.

int add(int a,int b){ return a+b; }

int add(int a,int b,int c){ return a+b+c; }

constexpr int square(int x){ return x*x; }

OOP — Classes & Objects

class Point{

public: int x,y; Point(int x=0,int y=0): x(x), y(y) {} void move(int dx,int dy){ x+=dx; y+=dy; } };

Point p(1,2); p.move(3,4);

Key: encapsulation, access specifiers (public, private, protected), constructors, destructors.

Inheritance & Polymorphism

class Base{ public: virtual void f(){ std::cout<<"Base\n"; } virtual ~Base(){} };

class Derived: public Base{ public: void f() override { std::cout<<"Derived\n"; } };

Base* b = new Derived(); b->f(); // calls Derived::f

Memory, Pointers & References

References (int &r = x) vs pointers (int *p). Prefer references when possible. Use smart pointers (unique_ptr, shared_ptr) instead of raw new/delete.

int x=10;

int &r = x; // alias int *p = &x; // pointer

Design Patterns & Best Practices

  • Rule of Three / Five / Zero
  • Prefer RAII, smart pointers
  • Minimize use of raw pointers; prefer references
  • Use const correctness and noexcept where possible
